## Detection Patterns

### Merchant Name Variations

Many services appear with variations in bank statements:

**Apple Services:**
- `APPLE.COM/BILL` — could be Apple Music, iCloud, Apple TV+, or App Store
- Need to check amount to determine which service

**Google Services:**
- `GOOGLE *YouTube` — YouTube Premium or YouTube Music
- `GOOGLE *Storage` — Google Drive/Google One
- `GOOGLE *Cloud` — Google Cloud Platform

**Amazon Services:**
- `AMZN Prime` — Amazon Prime (includes Prime Video)
- `AMZN Audible` — Audible subscription
- `AMZN AWS` — AWS cloud services

### Amount-Based Detection

When merchant name is ambiguous, use amount to identify service:

**Apple.com/bill amounts:**
- $0.99/month → iCloud 50GB
- $2.99/month → iCloud 200GB
- $4.99/month → Apple Music (student)
- $6.99/month → Apple TV+
- $9.99/month → iCloud 2TB or Apple Music (individual)

**Google amounts:**
- $1.99/month → Google One 100GB
- $2.99/month → Google One 200GB
- $9.99/month → Google One 2TB
- $11.99/month → YouTube Premium

### Frequency Patterns

**Monthly:** Most subscriptions charge monthly
**Quarterly:** Some business tools (e.g., certain SaaS)
**Annual:** Often discounted (e.g., Amazon Prime $139/year)

## Custom Keywords

If user has custom subscriptions not in this list, add them during audit:

```
Unknown recurring charge detected:
Merchant: "ACME CORP"
Amount: $29.99/month
Frequency: Monthly (3 months detected)

Action: Flag for user review in audit report
```

## False Positives to Avoid

**Not subscriptions:**
- One-time purchases from same merchant
- Irregular amounts (e.g., utility bills with variable usage)
- Merchant names that appear monthly but are different services (e.g., "Amazon" for shopping vs "AMZN Prime")

**Detection rule:**
- Must appear 2+ times
- Amount must be consistent (within 10% variance)
- Frequency must be regular (monthly/quarterly/annual)
